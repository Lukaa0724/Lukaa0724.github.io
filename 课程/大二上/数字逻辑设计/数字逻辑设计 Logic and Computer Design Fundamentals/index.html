    Chapter 3 Combinational Logic Design

    How to define? A combinational circuit consists of logic gates whose output is a function of only the present input. 输出只和当前的输入有关，不具有存储功能

    hierarchical design

    把函数分成小块 $$ A = B + C \ D = \overline{B + C} $$ 我们可以写成 $$ T = B+ C \ A = T\ D = \overline{T} $$ 分解出来的叫做的block(块) 没被分解的叫做（primitive block）原始块 包括分解的块在内的所有块的集合构成一个层次结构（hierarchy）

    举个例子： 用于9个输入奇偶树的分层设计 • 顶层：9个输入，一个输出 • 第二层：两个级别中的四个3位奇偶树 • 第三层：两个2位异或函数 • 原始块：四个2输入NAND门 • 设计需要32个2输入NAND门，即4 ╳ 2 ╳ 4 = 32个门

    Something about Integrate Circuits

    • Fan-in: the number of inputs available on a gate
    • Fan-out: the number of standard loads driven by a gate output
    • Logic Level: the signal value range for 1 and 0 on the inputs and 1 and 0 on the outputs
    • Noise Margin: the maximum external noise voltage superimposed on a normal input value that will not cause an undesirable change in the circuit output
    • Propagation Delay: the time required for a change in the value of a signal to propagate from an input to an output（传播延迟）
    • Power Dissipation – the amount of power drawn from the power supply and consumed by the gate 功耗

    Fan-in and Fan-out

    Fan-in : 一个门可以接受的最大输入的数量

    Fan-out: 不会降低操作的条件下，门输出的最大负载（门输出可以连到几个输入中）

    Transition time:the time required for the gate output to change from High to Low

    最大Fan-out也可以这样理解：使得Transition Time不会超过最大过渡时间的最大负载数量

    两种Delay Models

    • Transport delay(传输延迟)：输入改变后 过了固定的时间 输出改变 • Inertial delay（惯性延迟）：输入达到一定的强度 一定的时间 输出才会变化 也就是说Inertial delay可以屏蔽比较窄小的脉冲

    arithmetic calculations

    两个12bit的数相加 如果使用真值表 \(2^{12}\)恐怖如斯 太过复杂没有必要 所以引入加法器 半加器\(X+Y = C:S\) 真值表易得 \(\(\begin{aligned}\mathbf{S}&=\mathbf{X}~\overline{\mathbf{Y}}+\overline{\mathbf{X}}~\mathbf{Y}=\mathbf{X}\oplus\mathbf{Y}\\\mathbf{S}&=(\mathbf{X}+\mathbf{Y})~(\overline{\mathbf{X}}+\overline{\mathbf{Y}})\end{aligned}\)\) and \(C=XY\)这都很简单 全加器和半加器类似 只不过全加器需要加上前面的进位 \(\(\mathbf{S}=\mathbf{X}\oplus\mathbf{Y}\oplus\mathbf{Z}\)\) \(\(\mathbf{C}=\mathbf{X}\mathbf{Y}+(\mathbf{X}\oplus\mathbf{Y})\mathbf{Z}\)\) \(XY\) 叫做进位生成 Carry generate \(X\oplus Y\) 叫做进位传播 Carry propagate 普通的加法器存在一个问题，就是从开始传播到最后需要花费很长时间 效率太低 这个地方很容易混淆 AB都是1 才会生成进位 所以AB是生成函数 因此另一个就是传播函数 \(\(\begin{aligned}\color{blue}{\mathrm{P_i=A_i\oplus B_i}}&&\color{blue}{S_i=P_i\oplus C_i}\\\color{blue}{G_i=A_iB_i}&&\color{blue}{C_{i+1}=G_i+P_iC_i}\end{aligned}\)\) 通过上边的公式，我们很容易推导出式子，问题在于 想知道\(C_{i+1}\)必须知道\(C_i\)浪费时间和效率 所以需要不断迭代替换 \(\(\begin{aligned} &C_1 \begin{aligned}=\mathbf{G_0}+\mathbf{P_0}\mathbf{C_0}\end{aligned} \\ &C_2 \begin{array}{l}=\mathbf{G_1}+\mathbf{P_1}\mathbf{C_1}=\mathbf{G_1}+\mathbf{P_1}(\mathbf{G_0}+\mathbf{P_0}\mathbf{C_0})\\=\mathbf{G_1}+\mathbf{P_1}\mathbf{G_0}+\mathbf{P_1}\mathbf{P_0}\mathbf{C_0}\end{array} \\ &C_3 =\mathbf{G_2}+\mathbf{P_2}\mathbf{C_2}=\mathbf{G_2}+\mathbf{P_2}(\mathbf{G_1}+\mathbf{P_1}\mathbf{G_0}+\mathbf{P_1}\mathbf{P_0}\mathbf{C_0}) \\ &=\color{blue}{\mathbf{G}_2+\mathbf{P}_2\mathbf{G}_1+\mathbf{P}_2\mathbf{P}_1\mathbf{G}_0+\mathbf{P}_2\mathbf{P}_1\mathbf{P}_0\mathbf{~C}_0} \\ &\text{C4} =\mathbf{G}_3+\mathbf{P}_3\mathbf{C}_3=\mathbf{G}_3+\mathbf{P}_3\mathbf{G}_2+\mathbf{P}_3\mathbf{P}_2\mathbf{G}_1+\mathbf{P}_3\mathbf{P}_2\mathbf{P}_1\mathbf{G}_0 \\ &\color{red}{+P_3P_2P_1P_0C_0} \end{aligned}\)\) 最终就可以得到下面的式子 时间复杂度从\(O(N)\)降低到了\(O(1)\)
